How it works

01 // BOOT SEQUENCE
STEP 01

Run the harness

One npm start. The connector harness comes up on loopback with the Markdown brain already enabled.

cd packages/harness && npm start
STEP 02

Launch the orb

A frameless, always-on-top Tauri orb lands on your desktop. It reads voxa-config.json and finds the harness.

cd packages/orb && npm run tauri dev
STEP 03

Tap and talk

Open mic, low latency, barge-in — interrupt it mid-sentence. The model calls tools; the tools run on your machine.

“What’s on the Hacker News front page?”

02 // MEMORY

The local brain

Voxa searches, reads, and saves a folder of plain Markdown notes — by voice, offline, no API key. Point it at an Obsidian vault and talk to notes you already have.

  • ▸memory_search · memory_read · memory_save · memory_list

03 // SELF-EXTENSION

The Forge

Ask for a connector and Voxa builds it — from a safe, declarative, data-only HTTP spec. It never executes generated code. Great for any public REST API; the new tools are live the next time you tap the orb.

spec → validate → write. No eval, ever.
