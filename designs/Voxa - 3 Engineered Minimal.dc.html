03 — MEMORY OFFLINE · NO API KEY · PLAIN FILES

The local brain

Voxa searches, reads, and saves a folder of plain Markdown notes by voice. Point brainDir at an Obsidian vault and talk to the notes you already keep. They stay what they are — files on disk, yours to grep.

memory_searchfind notes by content memory_readread one aloud memory_savewrite a new .md file memory_listwhat's in the folder
TRANSCRIPT — SPECIMENLOCAL SESSION
YOU“Remember the standup moved to 10am.” VOXA“Noted.” → memory_save · standup.md ⋯LATER YOU“What time is standup?” VOXA“Standup’s at 10.”
04 — SELF-EXTENSION DECLARATIVE SPEC · SSRF-GUARDED · NO EVAL

The Forge

Voxa builds its own connectors, by voice. Ask for one and the forge writes it from a declarative, data-only HTTP spec — it never executes generated code. Works for any public REST API; the new tools are live on the next session.

spec → validate → write. No eval in the path.

PROCEDURE — FORGE RUN№ 04.1
“Build me a connector for the OpenWeather API.” → forge.plan declarative HTTP spec → forge.validate schema + SSRF guard → forge.write connectors/openweather/ ✓ tap the orb to reload — tools are live
